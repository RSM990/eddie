# eddie — Docs Maintenance

**Last Updated:** June 5, 2026 (starter)

How the docs stay current — same rhythm as the other projects.

| Doc | Update when |
|-----|-------------|
| planning/CURRENT_SPRINT.md | Every sprint / when the active sprint changes |
| current/CURRENT_STATUS.md | After each sprint or milestone |
| decisions/DECISIONS.md | When a decision is made (append, never rewrite history) |
| planning/ROADMAP.md | When phases complete or priorities shift |
| technical/V2_SCHEMA_CONTRACT.md | Whenever a consumer changes a shared table eddie writes |
| technical/ARCHITECTURE.md, TECH_STACK.md | When structure or dependencies change |
| foundation/VISION.md | Rarely — only on a real change of purpose |

## Conventions

- On sprint completion: archive `CURRENT_SPRINT.md` → `archive/PHASE<n>_SPRINT_PLAN.md`, write the next one, update CURRENT_STATUS + ROADMAP.
- Keep `V2_SCHEMA_CONTRACT.md` in sync with TheWAC in the same change that touches a shared table.
